# Fluent Mica Website Redesign — Pictelio 静态营销页重构

> **设计文档** · 2026-06-28
> 基于 Fluent Design 2 语言对 `website/index.html` 进行全面视觉重构

---

## 1. 目标

将现有的 Pictelio 静态营销着陆页从当前的 Fluent Design 基础实现升级为完整的 **Fluent Mica** 视觉体系，解决"太普通、不好看"的问题，同时避免 AI 产品常见的紫蓝渐变/发光风格。

## 2. 设计语言

| 维度     | 选择                             | 说明                     |
| -------- | -------------------------------- | ------------------------ |
| 体系     | **Fluent Design 2** (Windows 11) | Mica + Acrylic 材质体系  |
| 风格定位 | 原生质感、通透、高级             | 非 AI/非极简/非插画      |
| 核心理念 | 模拟 Windows 11 桌面级体验       | 让网页像原生应用一样精致 |

## 3. 布局结构

```
┌─────────────────────────────────────────────┐
│  HEADER — Sticky + Acrylic 背景模糊          │
├─────────────────────────────────────────────┤
│  HERO — 分割式                              │
│  左: 品牌 + 标题 + CTA                      │
│  右: Win11 窗口 + 手机模拟器                 │
│  背景: Mica 多层模糊 + 渐变底层              │
├─────────────────────────────────────────────┤
│  FEATURES — 2×2 Reveal 卡片网格              │
├─────────────────────────────────────────────┤
│  SCREENSHOTS — 横向轮播 + 手机框架           │
├─────────────────────────────────────────────┤
│  DOWNLOAD — 两张分层卡片 (GH + F-Droid)      │
├─────────────────────────────────────────────┤
│  DISCLAIMER + FOOTER — Acrylic 底部          │
└─────────────────────────────────────────────┘
```

## 4. 各部分详细设计

### 4.1 Header（全局导航）

- **位置**：`position: sticky; top: 0; z-index: 100`
- **背景**：Acrylic 毛玻璃（`backdrop-filter: blur(24px)`），半透明底 + 1px 底部描边
- **内容**：Logo + 品牌名 | 导航链接（功能 / 下载 / 隐私 / GitHub）
- **交互**：导航链接悬停时下划线展开动画（`transform: scaleX`），当前页面高亮

### 4.2 Hero 区域

- **背景层（Mica 模拟）**：
  - 最底层：`radial-gradient` 从右上角散开的柔和的蓝色渐变（`#0078d4` → `transparent`）
  - 上层：大面积 `backdrop-filter: blur(16px)` 模拟 Mica 材质对桌面壁纸的模糊效果
  - 实际实现：一个伪元素覆盖层带 `backdrop-filter`，叠加在渐变背景上
- **内容层（Acrylic 面板）**：
  - Hero 内容区域包裹在 `rgba(255,255,255,0.5)` 背景 + `backdrop-filter: blur(40px)` 的面板中
  - 1px 半透明边框（亮色：`rgba(0,0,0,0.06)`，深色：`rgba(255,255,255,0.08)`）
  - 圆角：`16px`
  - 阴影：双层阴影（Windows 11 风格），环境阴影 + 方向阴影

- **左侧内容**：
  - Logo SVG（现有画笔 P Logo）：96px 尺寸，`drop-shadow` 增强
  - 标题：`clamp(48px, 5vw, 80px)`，字重 700，品牌色强调 "Pictelio"
  - 副标题：`clamp(18px, 1.5vw, 24px)`，次要文字色
  - CTA 主按钮：Acrylic 毛玻璃填充 + 品牌色，`borderRadius: 10000px`
  - CTA 次按钮：透明描边

- **右侧 Windows 窗口模拟器**：
  - 一个 `div` 模拟 Windows 11 窗口：标题栏（三按钮 + 窗口标题）+ 内容区
  - 标题栏：Acrylic 效果，含最小化/最大化/关闭按钮（红黄绿三色圆点）
  - 内容区：深色背景，嵌入手机 SVG 框架展示 App 主界面截图
  - 窗口下方：模拟任务栏（居中开始菜单图标 + 右侧托盘时间）

### 4.3 Features 特性卡片

- **布局**：CSS Grid `2×2`（桌面），`1×4` 垂直（手机）
- **卡片样式**：
  - 背景：`rgba(255,255,255,0.4)` + `backdrop-filter: blur(20px)`
  - 圆角：`12px`，1px 半透明边框
  - 阴影：Windows 11 风格的双层阴影
  - 悬停 Reveal：卡片边缘出现跟随鼠标的径向渐变高亮（`radial-gradient` via JS mousemove）
  - 悬停上浮：`transform: translateY(-2px)` + 阴影加深
- **图标**：Fluent UI System Icons 风格的描边 SVG（28px 容器，品牌色）
- **内容**：标题（`18px` 字重 600）+ 描述（`14px` 字重 400）

### 4.4 截图画廊

- **容器**：横向滚动（`overflow-x: auto; scroll-snap-type: x mandatory`）
- **手机框架**：每个截图用 SVG 手机外壳包裹，带 Acrylic 风格投影
- **交互**：
  - 滚动吸附（`scroll-snap-align: center`）
  - 底部圆点导航指示器
  - 可选自动轮播（`setInterval` 滚动，悬停暂停）
- **截图内容**：使用现有的 4 张 App 截图（推荐流、作品详情、设置面板、登录页）

### 4.5 下载区域

- **布局**：Flex 双列（桌面），单列堆叠（手机）
- **下载卡片**：
  - GitHub 卡片（推荐）：Acrylic 背景，徽章 "推荐"，主按钮品牌色填充
  - F-Droid 卡片（即将上架）：半透明背景，徽章 "即将上架"，按钮禁用态
- **信任信号**：GitHub 卡片底部展示 APK 的 SHA-256 指纹（前 8 字符 + 「复制」按钮）
- **按钮样式**：与 Hero CTA 统一

### 4.6 声明 & Footer

- **声明区域**：Acrylic 面板 + 警示图标 + 排版与现有一致
- **Footer**：Acrylic 背景，简约导航链接 + 版权信息

## 6. 动效方案

| 元素        | 动效                              | 实现                                    |
| ----------- | --------------------------------- | --------------------------------------- |
| Reveal 高亮 | 鼠标移动时卡片边缘径向渐变高亮    | CSS `radial-gradient` + JS mousemove    |
| 卡片浮入    | 滚动到视口时卡片从下方浮入 + 渐显 | CSS `@keyframes` + IntersectionObserver |
| 导航下划线  | 悬停时从中间展开                  | `transform: scaleX` 过渡                |
| 按钮反馈    | 按压时缩放 `0.98` + 阴影变化      | CSS `transition`                        |
| 截图轮播    | 自动滚动 + 手动拖拽               | CSS `scroll-snap` + JS                  |
| 滚动渐入    | 各节进入视口时透明度 0→1 + 位移   | IntersectionObserver + CSS transition   |
| 窗口模拟    | 关闭按钮悬停时发光                | CSS transition                          |

## 7. 技术实现

- **技术栈**：纯 HTML + CSS + 少量 Vanilla JS（无框架依赖）
- **CSS 变量**：复用现有 Fluent Design token 体系（扩展 Mica/Acrylic 相关变量）
- **响应式断点**：
  - 桌面：`≥ 768px` — 布局就绪
  - 平板：`≥ 480px` — 2 列特性
  - 手机：`< 480px` — 单列堆叠
- **性能目标**：
  - 首屏内容 < 50KB CSS（内联关键样式）
  - 无外部字体依赖（系统字体栈）
  - 所有图片 WebP 格式
- **浏览器兼容**：现代浏览器（Chrome/Firefox/Safari/Edge 最近 2 个主版本）

## 8. 与现有设计的差异

| 项目      | 当前            | 新设计                          |
| --------- | --------------- | ------------------------------- |
| 背景      | 纯色 + 单渐变   | Mica 多层模糊 + 混合渐变        |
| Hero 面板 | 无独立面板      | Acrylic 毛玻璃面板 + Win11 窗口 |
| 卡片效果  | 纯色 + 简单阴影 | Acrylic 毛玻璃 + Reveal 高亮    |
| 截图展示  | 静态伪元素模拟  | 真实横向轮播 + 手机框架         |
| 动效      | 基本 hover      | 滚动渐入 + Reveal + 轮播        |
| 排版      | 常规尺寸        | 超大标题 + 更清晰的层级         |
| 信任信号  | 无              | 新增 SHA-256 指纹展示           |

## 9. 范围与约束

- **范围**：仅重构 `website/index.html`，不涉及 `website/privacy-policy.html` 或其他文件
- **不涉及**：主应用 UI、Capacitor 配置、Android 原生代码
- **部署**：通过 GitHub Pages 自动部署（gh-pages 分支），构建流程不变
- **维护**：重构后仍为单 HTML 文件，便于直接编辑和部署
